deployAppSettings

  • deployAppSettings(apps: AppInfoRequest | AppInfoRequest[], revert?: undefined | true | false): Promise<void>
    • Defined in api/app-api.ts:172

    Updates the settings of a pre-live App to the live App.

    Using this API is the same as clicking on "Update App" in the kintone App's settings. Changes made to pre-live settings will not be deployed to the live app until this API is used.

    This API can also be used to revert the pre-live settings back to the current live app settings. This would be the same as clicking on "Discard Changes" in the kintone App's settings.

    If multiple apps are specified in this API, and one of the updates to an App fails, all of the specified Apps will rollback to the state before this API was run.

    App Management Permissions are needed. API Tokens cannot be used with this API.

    see

    https://developer.kintone.io/hc/en-us/articles/115004881348

    Parameters

    • apps: AppInfoRequest | AppInfoRequest[]

      The App or list of Apps to deploy the pre-live settings to the live Apps. The Maximum limit is 300. If Apps are being deployed to Guest Spaces, Apps can only be deployed to the same Guest Space.

    • Optional revert: undefined | true | false

      Specify "true" to cancel all changes made to the pre-live settings. The pre-live settings will be reverted back to the current settings of the live app.

    Returns Promise<void>

getApps

  • getApps(appIds?: number[], codes?: string[], name?: undefined | string, spaceIds?: number[], limit?: undefined | number, offset?: undefined | number): Promise<App[]>
    • Defined in api/app-api.ts:85

    Gets general information of multiple Apps, including the name, description, related Space, creator and updater information.

    Permissions to view the Apps are needed. API Tokens cannot be used with this API.

    see

    https://developer.kintone.io/hc/en-us/articles/115005336727

    Parameters

    • Optional appIds: number[]

      The App IDs. Up to 100 IDs can be specified.

    • Optional codes: string[]

      The App Code. Up to 100 App Codes can be specified. Each App Code must be between 1 to 64 characters. An exact match search will be used, and will be case sensitive.

    • Optional name: undefined | string

      The App Name. A partial search will be used, and the search will be case insensitive. The localized name of the App in the user's locale will also be included in the search.

    • Optional spaceIds: number[]

      The Space ID of where the App resides in. Up to 100 IDs can be specified.

    • Optional limit: undefined | number

      The number of Apps to retrieve. Must be between 1 and 100. If nothing is specified, it will default to 100.

    • Optional offset: undefined | number

      The number of retrievals that will be skipped. Must be between 0 and 2147483647. If nothing is specified, it will default to 0.

    Returns Promise<App[]>
